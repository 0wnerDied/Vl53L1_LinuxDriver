Detailed Description

stmvl53l1 can be controled via a set of exposed as sysf attribute under /sys/class/inputx/attribe_name where x is the input device number associated with the vl53l1 device . X is a dynamic number that depend on input device present in the system it can changes, depending order of driver loaded, hot plug device like usb mice and keyboard.

sysfs attribute "roi" [rd/wr]

  • read show the current user customized roi setting
  • write set user custom roi, it can only be done while not ranging.

syntax for set input roi

  • "[tlx tly brx bry,]\n" repeat n time require will set the n roi
  • "\n" will reset
Warning
roi coordinate is not image x,y(down) but euclidian x,y(up)
roi validity is only check at next range start
user is responsible to set appropriate number an roi before each mode change
Note
roi can be return to default by setting none ""
1 >#to set 2x roi
2 >echo "0 15 15 0, 0 8 8 0" > /sys/class/input6/roi
3 >echo $?
4 0
5 >cat /sys/class/input6/roi
6 "0 15 15 0,0 8 8 0"
7 #to cancel user define roi"
8 >echo "" > /sys/class/input1/roi
9 >echo $?
10 0
11 >echo "1" > /sys/class/input6/enable_ps_senspor
12 #try to set roi while ranging
13 >echo "0 15 15 0, 0 8 8 0" > /sys/class/input6/roi
14 [58451.912109] stmvl53l1_store_roi: cant set roi now
15 >echo $?
